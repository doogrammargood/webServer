Projects

My current project is a multidimensional Tic-Tac-Toe game written in Javascript.
The code


Description

  • Players X and O (gold and silver) take turns marking squares
  • A player can only play on the allowed board (purple), which is determined by the previous moves
  • A player captures a board by getting 3 in a row. If the other player gets 2 sets of 3 in a row, he may steal the board.
  • The game is won when the largest board is captured.
  • If a player is forced to play on a full board he/she may play anywhere within the larger board containing this full board.
  • The Toggle Size button creates a new game with a different size.
  • The Add Bot creates a bot which uses alpha beta/pruning with a depth search up to 5.
  • You can press the Add Bot button again to see the bot play itself. (this takes a little while on the largest board)
  • For some reason, this doesn't work on mobile. Possibly due to CSS grid?
  • The Undo Move button undoes the last move. If you are playing a bot, you'll have to click it twice to undo the bot's move and your last move.
  • Through development, the undo move button has been buggy. It had better work now, because the bot relies on this function.

My next project will be to create a program which takes a video of a person juggling balls and uses some computer vision techniques to determine the siteswap. Jumbled code can be found here. My plan is the following:

  1. Apply a background filter based on motion.
  2. Use Hough Circle detection to find circles, which are hopefully mostly balls.
  3. Plot these balls and fit parabolas to them.
  4. Use wishful thinking on throws that are too fast to be captured on camera.
Interestingly, apart from the wishful thinking, there's no neural networks or probabilistic machine learning. When this plan fails, I'll try and train a neural network to watch the juggler's hands to handle the vamps.
